
# Environment Setup (env-setup)

Этот документ описывает **пошаговую настройку рабочего окружения** для репозитория **Classroom**.
Он предназначен для обучения, ведения заметок и практики и **не привязан к конкретному языку программирования**.

Основная идея — создать **универсальную, понятную и воспроизводимую среду**, в которой удобно:

* читать и писать Markdown
* работать с Git
* постепенно подключать новые инструменты

---

## 1. Общий подход к окружению

Окружение — это не только установленные программы, но и:

* структура папок
* способы работы с материалами
* привычки и соглашения

В Classroom окружение должно быть:

* простым
* расширяемым
* не перегруженным на старте

---

## 2. Установка и настройка Git

Git — основной инструмент взаимодействия с репозиторием.

### Проверка наличия Git

Откройте терминал и выполните:

```bash
git --version
```

Если команда не найдена, Git необходимо установить.

---

### Базовая настройка Git

После установки выполните первоначальную настройку:

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "you@example.com"
```

Проверить настройки:

```bash
git config --list
```

---

### Клонирование репозитория

Перейдите в каталог, где вы храните учебные проекты, и выполните:

```bash
git clone https://github.com/Eldemont357/Classroom.git
cd Classroom
```

---

### Обновление материалов

Для получения обновлений из репозитория:

```bash
git pull
```

Эту команду рекомендуется выполнять регулярно.

---

## 3. Работа с терминалом

Терминал используется для:

* навигации по проекту
* работы с Git
* запуска скриптов

### Базовые команды

```bash
ls        # список файлов (Windows: dir)
cd setup  # переход в папку
cd ..     # переход на уровень выше
pwd       # текущий путь
```

---

### Рекомендуемый рабочий сценарий

* Открыть терминал в корне `Classroom`
* Работать внутри одной директории
* Не выполнять команды вслепую — понимать, что делает каждая из них

---

## 4. Текстовый редактор и Markdown

Все материалы Classroom хранятся в формате **Markdown** (`.md`).

### Почему Markdown

Markdown позволяет:

* писать структурированные заметки
* легко читать текст
* хранить знания в системе контроля версий

---

### Использование VS Code (рекомендуется)

Открыть проект:

```bash
code .
```

Это откроет папку Classroom целиком.

---

### Полезные сочетания в VS Code

* `Ctrl + Shift + V` — предпросмотр Markdown
* `Ctrl + P` — быстрый поиск файла
* `Ctrl + `` — встроенный терминал

---

### Альтернативные редакторы

Допустимо использовать:

* **Obsidian** — как базу знаний
* **Typora** — как визуальный редактор

Главное требование — корректная работа с Markdown.
